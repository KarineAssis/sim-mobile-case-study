O **SIM Mobile** foi uma iniciativa de transformação digital aplicada à manutenção industrial. A solução levou fluxos relevantes do sistema corporativo para dispositivos móveis, aproximando o registro das informações do local onde o trabalho era executado.

[Estudo de caso em português](CASE_STUDY_PT.md) · [Case study in English](CASE_STUDY_EN.md)

## Visão geral

| Início | Fluxos mobile | Usuários habilitados | Unidades acompanhadas |
|:---:|:---:|:---:|:---:|
| **2021** | **5+** | **148** | **3** |

> Os números comprovam o escopo de implantação e a liberação de acessos. Usuários habilitados não equivalem necessariamente a usuários ativos.

## O problema e a mudança

<div align="center">
  <img src="assets/processo-antes-depois.svg" alt="Comparação entre o processo anterior e o processo com o SIM Mobile" width="100%">
</div>

Antes do projeto, atividades realizadas no chão de fábrica ainda dependiam de computadores fixos em salas de apoio. Quando o acesso não era imediato, o registro podia acontecer somente depois da execução.

## Solução

O SIM Mobile foi desenvolvido como uma extensão dos fluxos mais relevantes do sistema corporativo, sem reproduzir integralmente a versão desktop.

| Fluxos disponibilizados | |
|---|---|
| Manutenção corretiva | Criação e atualização de ordens |
| Consulta de estoque | Manutenção preventiva |
| Consulta de apontamentos de mão de obra | Requisição de materiais |

## Minha atuação

Conduzi a iniciativa pelo lado do negócio, em interface com a equipe de Tecnologia da Informação.

**Necessidade operacional → requisitos funcionais → alinhamento com TI → testes → piloto → treinamento e implantação**

Minha responsabilidade foi garantir que a solução refletisse os processos reais da manutenção. O desenvolvimento técnico do software ficou sob responsabilidade da equipe de TI.

## Impacto documentado

| Mudanças observadas | Limites das evidências |
|---|---|
| Registro mais próximo da execução | Não comprova frequência diária de uso |
| Menor dependência de computadores fixos | Não comprova adoção por funcionalidade |
| Maior conexão entre execução e sistema | Não quantifica redução de tempo |
| Base mais favorável para dados atualizados | Não atribui ganho financeiro ou produtividade |
| Maior colaboração entre manutenção e tecnologia | Resultados limitados aos registros históricos disponíveis |
